# Supply Setu - Resume Points

**Role:** System Designer & Mobile App Developer  
**Tech Stack:** React Native (Expo), Node.js, Next.js, PostgreSQL, MongoDB, Redis, RabbitMQ, Socket.io, Solidity, Hardhat

### Option 1: Detailed & Impact-Driven (Recommended for System Design Roles)
* **Architected Event-Driven Microservices:** Designed and deployed a robust B2B supply chain platform utilizing Node.js and RabbitMQ, enabling asynchronous service communication that eliminated "ghost orders" through instant, distributed inventory synchronization.
* **Engineered Polyglot Data Strategy:** Implemented a highly scalable persistence layer combining PostgreSQL for ACID-compliant transactions (Auth/Orders), MongoDB for dynamic catalogs, and Redis for high-throughput, low-latency cart operations.
* **Developed Cross-Platform Mobile Apps:** Built feature-rich Vendor and Delivery applications using React Native (Expo), integrating condition-based live GPS tracking, localized state management, and secure JWT authentication via expo-secure-store.
* **Integrated Blockchain Traceability:** Designed a zero-trust immutable architecture using Solidity smart contracts (Hardhat) to generate "Digital Birth Certificates" for product tracing and tamper-proof reviews, abstracting gas fees for seamless UX.
* **Implemented Real-Time Market Infrastructure:** Integrated Socket.io to facilitate live market interactions, driving instantaneous communication for bulk order bidding and community negotiations.

### Option 2: Concise & Action-Oriented (Recommended for General Software Engineering Roles)
* Designed a hyper-local B2B supply chain ecosystem by architecting a scalable microservices backend (Node.js, RabbitMQ) and an API Gateway (Next.js) to securely route internal network traffic.
* Developed high-performance React Native mobile applications for vendors and delivery personnel, featuring atomic cart management, secure local storage, and real-time conditional GPS tracking.
* Optimized data retrieval and system reliability through a strategic polyglot persistence model utilizing PostgreSQL, MongoDB, and Redis to handle complex order lifecycles and real-time data caching.
* Integrated Ethereum smart contracts using Hardhat and Solidity to establish immutable product traceability and vendor reputation systems, enhancing platform trust without degrading user experience.

### Option 3: Mobile Apps & Frontend Focused
* Led the frontend mobile development of Vendor and Delivery apps utilizing React Native and Expo, delivering a seamless B2B sourcing experience with secure authentication and atomic cart features.
* Implemented complex tracking UI components that conditionally unlock live GPS mapping (Socket.io) based on collection status, reducing tracking overhead and improving UX.
* Collaborated on system architecture to integrate mobile clients seamlessly with an event-driven backend, ensuring real-time UI updates for order progression from pending to dispatch.
* Integrated web3 capabilities gracefully into mobile interfaces, exposing blockchain-verified origin stories and permanent reviews to users securely without requiring external crypto wallets.

---

### 💡 Tips for your resume:
- Tailor the bullet points based on the specific job description. Use **Option 1** for backend/system design heavy roles, and **Option 3** if you are applying for mobile-centric roles.
- Be prepared to discuss **RabbitMQ** (how you handled message delivery guarantees/failures) and **Database choices** (why you chose MongoDB for products vs. PostgreSQL for orders) during interviews, as these are strong talking points for system design!
